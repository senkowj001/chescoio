{% comment %} Checkout success status — HTMX-swappable fragment. The Stripe webhook is responsible for creating the local Order. The customer often lands here a beat before the webhook fires, so we poll the same endpoint until the Order exists or the poll budget runs out. Three states: - order present => receipt detail, no polling - timed_out=True => apologetic timeout, no polling, link to keep shopping - else => "processing" state that polls every 2s with an incrementing attempt counter; the view returns timed_out=True once attempt hits CHECKOUT_SUCCESS_MAX_POLLS The attempt counter is carried through the URL so the budget is enforced server-side. Client-side state would be lost across the HTMX outerHTML swap. {% endcomment %} {% load catalog_extras %}
